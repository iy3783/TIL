가상화의 종류
-서버 가상화 : 서버 운영체제 기반으로 하드웨어 논리적으로
구분 Hyper-v (server가상화)
-데스크탑 가상화 : 버츄얼 pc
-프리젠테이션 가상화 : 원격 pc 화면 공유
-응용 프로그램 가상화 : 
설치 파일 복사 + 응용프로그램 정보 os등록(레지스트리)
+ 등록된 정보들 응용 프로그램 실행될때 사용됨
=> 응용프로그램 가상화 sw가 os에 깔려있으면 
똑같은 어플도 동시에 실행 가능

2. hiper-visor 가상머신은 원래 못보는데 가상머신 컨트롤 가능하게
해주는 vmm 
: cpu 메모리 등을 포함한 하드웨어 자원들을 각 vm에 논리적으로 분할하여
할당

게스트 os 에서 커널 모드 인건 실제 cpu에서 실행 되어야함으로
hiper-visor같은 가상화 sw(RING1)를 통해서 전달되어야한다.
RING0(host os)에 들어가기위한 창구인 RING 1생긴다.

3. 전체 가상화 방식 :둘다 하이퍼 바이저 방식이다.
-모노리틱 하이퍼 바이저: vmware esxi
그냥 하이퍼바이저 위에 관리os, 각각의 vm이 있다.
하이퍼바이저에 하드웨어 드라이버가 있다.
 성능 조금더 좋음

-마이크로 커널 : hyper-V
모노리틱과 똑같이 생겼지만 하드웨어 접근 방식이 다르다.
드라이버를 관리 os에서 관리한다.(드라이버에 대한 모든 권한을
vm에서 사용가능하다.)
