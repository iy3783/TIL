2. 패스워드 방식의 특징과 단점
장점 : 다양한 분야에서 사용 가능 , 검증확실(모르면 인증x)모호성 없다.
관리 비용 적다.

단점: 누가 알기만 하면 인증 됌; 비밀번호 까먹기 쉽다.
brute force ,dictionary , trojan horse login program

3.메시지 출처 인증 기술
: 요즘은 데이터 비밀 전송 보다 데이터 인증에 더 주력하고 있다.
변형 없는 것을 보장 위해 메시지 무결성을 지키려고 한다.

메시지 인증 코드(MAC) : 무결성 확인하고 메시지 인증하는 기술
임의의 길이의 메시지로 부터 고정비트길이 출력하는 것은 해시함수랑 비슷
해시와의 차이점은 수신자 송신자가 key를 공유한다.

key+message 해시함수에 넣어서 MAC 만듬

4. 메시지 무결성 찾는 변조 감지 코드(MDC)
:  MAC과의 차이=> 
해시 해서 만든 고정 비트 값을 안전한 채널로 보내고 이를 비교하는게 MDC
공유키를 같이 보내서 공유키가 있어야만 판단되는게 MAC

7.  DOS 공격 종류(flooding)
-ping of death attack : ping 계속 보내서 서버 부하
-land attack : 패킷의 소스 des 모두 같게해서 안좋은 상태에 빠지게 함
-smurf attack : ip 위장
teardrop attack : 패킷들의 offset을 범위 넘게 해서 오류 빠지게 함

8. icmp ,udp , SYN flooding (tcp 3way handshaking 취약점:위조 IP로

9. IDS (intrusion detection system 침입 탐지 시스템): 
외부 침입이나 의심스러운 행위를 감시해서 조기 발견해서 담당자에게 알림
IDS 단점 : 근본적인 해결책은 아님 , 새로운 침입기법에 대한 즉각 반응 x
-Raw data collection
-data 가공 filtering
-침입 분석 / 탐지
 침입 system의 핵심 단계
 비정상적 사용 탐지+오용탐지,system,응용sw에러)
(hybrid 탐지 기법)
-관리자 보고

기능적 특성:
탐지 방법 : 행위 기반 , 지식 기반 (오용탐지(이미 등록된패턴과 비교) , 비정상 행위 탐지)
대응 방법 : 수동적 ,능동적
데이터 수집원 : 호스트 로그파일 , 네트워크 패킷

네트워크 기반 패킷 탐지가 더 싸다. HIDS는 추가적 HW 필요 , 구현어렵
but 호스트 기반은 내부 외부 전부 탐지
가능, 그리고 더 정확하다.
